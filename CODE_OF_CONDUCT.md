# SattLint Code of Conduct

SattLint is maintained as a public technical project. We want the repository, issues, reviews, and release discussions to stay respectful, constructive, and safe for contributors and users.

## Expected behavior

- Be respectful, patient, and specific when giving feedback.
- Focus on the code, documentation, or workflow rather than the person behind it.
- Share reproducible examples, logs, or command output when reporting problems.
- Protect privacy: do not post secrets, private source code, personal data, or machine-specific paths unless they are redacted first.
- Keep proposals and examples legal, safe, and appropriate for a public repository.

## Unacceptable behavior

- Harassment, threats, intimidation, or personal attacks.
- Discriminatory or demeaning language.
- Publishing private information without permission.
- Knowingly posting malicious payloads, exploit chains, or unsafe instructions outside the project's security-reporting path.
- Repeatedly derailing technical discussions after maintainers ask for a narrower or safer path.

## Scope

This code of conduct applies to repository issues, pull requests, code reviews, release notes, documentation discussions, and other project-managed collaboration spaces.

## Maintainer responsibilities

Maintainers may edit, hide, lock, or remove content that breaks these expectations. They may also pause participation or block accounts when needed to protect contributors, users, or the project.

## Reporting

For routine questions about project support, use the paths in [SUPPORT.md](SUPPORT.md).

Do not file a public issue for safety-sensitive or privacy-sensitive conduct concerns. If a report cannot be shared publicly, use the private maintainer contact path described in [SECURITY.md](SECURITY.md) and clearly label it as a conduct report rather than a vulnerability report.
